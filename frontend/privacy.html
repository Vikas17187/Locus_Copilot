Privacy Notice

This demo application keeps user data handling intentionally simple. Use this page to set expectations clearly when presenting to evaluators or stakeholders.

Data We Store

Account details required for login, your saved preferences, and optional saved searches or favorites. Contact form submissions are stored locally in browser storage in this demo setup.

How Data Is Used

Data is used only to authenticate users, personalize ranking behavior, and improve repeat session experience. No ad targeting or third-party analytics are enabled by default in this project.

Security Model

Passwords are hashed on the backend. Environment variables are used for sensitive values. For production, add HTTPS, rate limiting, and stricter access control before external release.

Retention and Deletion

As this is a demo, retention is minimal and operational. You can clear local browser storage at any time and reset server-side demo data directly from the database for testing cycles.
